# Бот для перевізників (канал + бот)

Повторює схему `MK Merchants`: оголошення публікуються в каналі з кнопкою,
перевізники клікають → бот у приватному чаті приймає два тарифи (з ПДВ / без ПДВ),
адмін отримує звіт з найкращою ціною.

---

## Функціонал

**Для вас (адмін):**

- `/new` — майстер створення оголошення (маршрут, вантаж, вага, дата, контакт).
  Бот публікує пост у каналі з inline-кнопкою «💵 Ввести вашу пропозицію».
- `/list` — список активних оголошень (🟢 відкрито / 🟡 в роботі) з кількістю пропозицій.
- `/report <id>` — повний звіт по оголошенню: усі пропозиції, сортування, найкращі ціни окремо з ПДВ і без ПДВ.
- `/close <id>` — закрити оголошення (у каналі статус змінюється з 🟢/🟡 на 🔴).
- `/reopen <id>` — повернути активний статус.

**Для перевізника:**

- Клік по кнопці в каналі → картка заявки у приватному чаті з ботом.
- Дві кнопки: «📝 Ввести тариф з ПДВ» / «📝 Ввести тариф без ПДВ».
- `/contact` — передати контактний номер адміну.
- `/reset` — вийти з режиму введення ціни.

**Автоматично:**

- Коли перша пропозиція надходить — статус оголошення змінюється 🟢 → 🟡.
- Адмін отримує миттєве сповіщення про кожну нову ціну.
- Кожна заявка перевізника має унікальний номер, прив'язаний до оголошення.

## Деплой на Railway (рекомендований шлях)

### 1. Підготовка

Переконайтесь, що у вас:

- [x] Канал створено і бот доданий туди адміністратором з правом «Публікувати повідомлення».
- [x] Токен бота з `@BotFather`.
- [x] `ADMIN_ID` (ваш Telegram ID, через `@userinfobot`).
- [x] `CHANNEL_ID` каналу (через `@username_to_id_bot`, починається з `-100`).
- [x] `BOT_USERNAME` — юзернейм бота без `@` (наприклад, `vzayemodiya_z_pereviznykom_bot`).

### 2. Завантажити код на GitHub

1. Створіть акаунт на [github.com](https://github.com), якщо ще немає.
2. Створіть новий **приватний** репозиторій (наприклад, `pereviznyk-bot`).
3. Завантажте всі файли з цієї папки в репозиторій:
   - Через веб-інтерфейс: «Add file → Upload files» → перетягніть файли.
   - Або через git: `git init && git add . && git commit -m "init" && git push`.

Важливо: не додавайте `.env` у репозиторій — він у `.gitignore`.

### 3. Розгорнути на Railway

1. Зареєструйтесь на [railway.app](https://railway.app) (вхід через GitHub).
2. На дашборді → **New Project** → **Deploy from GitHub repo** → оберіть `pereviznyk-bot`.
3. Railway автоматично збере проєкт (використає `railway.json` + `requirements.txt`).
4. Перейдіть у вкладку **Variables** і додайте:

   | Key | Value |
   |---|---|
   | `BOT_TOKEN` | токен з @BotFather |
   | `ADMIN_ID` | `772817` (ваш ID) |
   | `CHANNEL_ID` | `-1003958283733` (ваш канал) |
   | `BOT_USERNAME` | `vzayemodiya_z_pereviznykom_bot` (без @) |
   | `DB_PATH` | `/data/bot.db` |

5. Додайте **Volume** для збереження БД:
   - Вкладка **Volumes** → **+ New Volume** → Mount path: `/data`.
   - Без цього кроку при кожному деплої база обнулятиметься.

6. Натисніть **Deploy**. Через 1-2 хв у логах побачите `Старт бота…`.

### 4. Перевірка

У Telegram:
1. Надішліть боту `/start` зі свого акаунта → отримаєте адмін-панель.
2. Надішліть `/new` → пройдіть майстер → перевірте, що пост з'явився в каналі.
3. З іншого акаунта (або попросіть колегу) клікніть кнопку під постом → введіть тариф.
4. Перевірте, що вам прийшло сповіщення.
5. Надішліть `/report <номер>` → побачите звіт.

## Локальний запуск (для тестування)

```bash
# Клонувати або розархівувати папку
cd pereviznyk-bot

# Створити віртуальне середовище
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# Встановити залежності
pip install -r requirements.txt

# Скопіювати шаблон env і заповнити
cp .env.example .env
# відкрийте .env у редакторі, впишіть BOT_TOKEN тощо

# Завантажити env і запустити
export $(grep -v '^#' .env | xargs)      # Linux/Mac
python bot.py
```

На Windows (PowerShell) замість `export`:
```powershell
Get-Content .env | ForEach-Object {
  if ($_ -match '^\s*([^#=]+)=(.*)$') { $env:($Matches[1].Trim()) = $Matches[2].Trim() }
}
python bot.py
```

## Додавання ще одного адміністратора

У `bot.py` функція `is_admin` перевіряє лише `ADMIN_ID`. Щоб додати ще одного:

```python
ADMIN_IDS = {
    int(x.strip()) for x in os.getenv("ADMIN_ID", "0").split(",") if x.strip()
}

def is_admin(user_id: int) -> bool:
    return user_id in ADMIN_IDS
```

І в `.env` / Railway Variables:
```
ADMIN_ID=772817,123456789
```
