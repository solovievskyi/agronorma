# Бот для перевізників (канал + бот) — v3.1

Оголошення публікуються у каналі з кнопкою, перевізники реєструються і подають пропозиції у приватному чаті з ботом, адмін отримує повну картку по кожному + фінальний звіт із можливістю обрати переможця.

---

## Що нового у v3.1

- **Схвалення реєстрації адміном** — перевізник подає заявку (ПІБ, телефон, ЄДРПОУ), вона летить у приватний чат усім адмінам з кнопками `✅ Схвалити` / `❌ Відхилити`. Поки заявка не схвалена — перевізник не може подавати пропозиції. Це відсікає сторонніх людей.
- **Спрощена реєстрація** — прибрано обов'язкові поля «тонаж» і «тип авто» (вони були зайвими: кожне оголошення вже має свою вагу, і перевізники беруть лише те, що їм підходить). Заявка — 3 кроки замість 5.
- **Список очікуючих** — у головному меню адміна нова кнопка `📝 Реєстрації (N)` зі списком непідтверджених заявок.

## Що є у v3

- **Реєстрація перевізника** (ПІБ/назва, телефон, ЄДРПОУ) — обов'язкова перед першою пропозицією. У звіті адмін бачить професійну картку замість анонімного `@username`.
- **Валідація** ціни, телефону, ЄДРПОУ з поясненнями, що саме не так.
- **Логування** у файл `bot.log` + будь-яка необроблена помилка летить супер-адміну як `<pre>` з traceback.
- **Щоденний бекап БД** — о 09:00 за Києвом бот шле `bot.db` супер-адміну.
- **Кнопка «✅ Обрати переможця»** у звіті. Переможець отримує контакт замовника, решта — «дякуємо, цього разу не пройшла», оголошення автоматично закривається.
- **Редагування** оголошення (8 полів) через inline-меню, канал оновлюється миттєво.
- **Експорт в Excel** (.xlsx) одним натисканням — з усіма даними, включно з ЄДРПОУ і типом авто.
- **Автозакриття** через `AUTO_CLOSE_HOURS` годин (за замовчуванням 48) + нагадування перевізникам за 2 год до закриття.
- **Розсилка** всім перевізникам — одне повідомлення кнопкою «📢 Розсилка».
- **Чорний список** — блокувати недобросовісних перевізників з причиною.
- **Історія перевізника** — у кожній картці видно `X перемог / Y заявок`.
- **Фото вантажу** — опціональний крок у майстрі, публікується разом з постом.
- **Фільтри** у списку — Усі / 🟢 / 🟡 / 🔴.
- **Статистика** — за 7 і 30 днів: оголошення, середня ціна, топ маршрути, топ перевізники.

## Головне меню адміна

```
┌─────────────────────────────────────────┐
│ 🆕 Нове оголошення │ 📋 Активні         │
│ 📊 Статистика │ 📝 Реєстрації (N)       │
│ 👥 Адміни │ 📢 Розсилка                  │  ← лише головний адмін
└─────────────────────────────────────────┘
```

Число `(N)` біля кнопки «Реєстрації» — скільки заявок очікують підтвердження. Кнопка з'являється з лічильником, коли є незавершені заявки.

Під кожним оголошенням у списку — inline-кнопки:
`📊 Звіт` · `📥 Excel` · `✏️ Редагувати` · `🔴 Закрити`/`🟢 Відкрити`

У звіті під кожною пропозицією:
`✅ Обрати переможцем` · `🚫 В чорний список`

## Меню перевізника

До реєстрації — форма з 3 кроків (ПІБ, телефон, ЄДРПОУ) → заявка відправляється адміну на перевірку. Поки адмін не схвалив — перевізник бачить повідомлення `⏳ Ваша заявка на реєстрацію ще на перевірці` при будь-якій дії.

Після схвалення:

```
┌───────────────────────┐
│ 👤 Мій профіль        │
│ ❓ Як користуватись    │
└───────────────────────┘
```

При натисканні кнопки під оголошенням у каналі — карта заявки:

```
┌─────────────────────────────────────┐
│ 📝 Тариф з ПДВ │ 📝 Тариф без ПДВ   │
│ 📱 Передати контакт                 │
│ ❌ Скасувати                        │
└─────────────────────────────────────┘
```

## Деплой на Railway

### 1. Передумови

- Канал створено, бот доданий як адміністратор з правом «Публікувати повідомлення».
- Токен з `@BotFather`.
- `ADMIN_ID` — ваш Telegram ID (через `@userinfobot`).
- `CHANNEL_ID` каналу (через `@username_to_id_bot`, починається з `-100`).
- `BOT_USERNAME` — юзернейм бота без `@`.

### 2. GitHub

1. Створити приватний репо.
2. Завантажити всі файли (окрім `.env` — він у `.gitignore`).

### 3. Railway Variables

| Key | Value | Обов'язково |
|---|---|---|
| `BOT_TOKEN` | токен з @BotFather | ✅ |
| `ADMIN_ID` | `772817` | ✅ |
| `CHANNEL_ID` | `-1003958283733` | ✅ |
| `BOT_USERNAME` | `vzayemodiya_z_pereviznykom_bot` | ✅ |
| `DB_PATH` | `/data/bot.db` | ✅ (щоб Volume підхопився) |
| `AUTO_CLOSE_HOURS` | `48` | ні, дефолт 48 |
| `REMINDER_BEFORE_HOURS` | `2` | ні, дефолт 2 |
| `BACKUP_HOUR_KYIV` | `9` | ні, дефолт 09:00 |
| `LOG_FILE` | `/data/bot.log` | ні |

### 4. Volumes

У Railway → Volumes → `+ New Volume` → Mount path `/data`. Без цього бекапи перестануть мати сенс після деплою.

### 5. Перевірка

1. `/start` → ви адмін → головне меню.
2. `🆕 Нове оголошення` → майстер 9 кроків → `✅ Опублікувати`.
3. З іншого акаунта клік під постом → проходимо реєстрацію (3 кроки) → заявка летить адміну.
4. Адмін у приватному чаті бачить картку заявки з кнопками `✅ Схвалити` / `❌ Відхилити`.
5. Після схвалення перевізник може подавати ціну. Адмін отримує сповіщення + бачить у звіті.
6. `📥 Excel` — скачується файл.
7. `✅ Обрати переможця` — всі повідомлення надсилаються автоматично.

## Керування адмінами

Лише головний адмін (`ADMIN_ID` з env):

- `👥 Адміни` → `➕ Додати адміна` → переслати повідомлення від користувача або ввести user ID.
- `🗑 Ім'я` — видалити.
- `/addadmin <user_id>`, `/rmadmin <user_id>` — через команди.

Головного адміна видалити не можна.

## Схвалення реєстрації (v3.1)

1. Перевізник натискає кнопку під постом у каналі вперше → отримує форму реєстрації (ПІБ, телефон, ЄДРПОУ).
2. Після 3-го кроку бот відповідає: `⏳ Заявку відправлено на розгляд. Очікуйте підтвердження адміністратора.`
3. Усі адміни отримують у приват повідомлення:
   ```
   📝 Нова заявка на реєстрацію
   Користувач: @username
   ПІБ: ТОВ «Приклад»
   Телефон: +380501112233
   ЄДРПОУ/ІПН: 12345678
   [✅ Схвалити]  [❌ Відхилити]
   ```
4. Натискання `✅ Схвалити` — перевізнику летить повідомлення `✅ Вашу реєстрацію підтверджено. Тепер ви можете подавати пропозиції.`
5. Натискання `❌ Відхилити` → адмін вводить причину → перевізник отримує повідомлення з причиною і більше не може подати пропозицію (іде в blacklist автоматично).
6. Якщо два адміни одночасно натиснуть кнопки — перший виграє, другий побачить alert `Заявка вже оброблена іншим адміном`.

Переглянути всі заявки, що чекають: кнопка `📝 Реєстрації` у меню адміна або команда `/pending`.

## Blacklist

У звіті під кожною пропозицією → `🚫 В чорний список` → ввести причину. Перевізник побачить повідомлення з причиною при наступній спробі й не зможе подати пропозицію.

Розблокувати — там само: `↩️ Розблокувати`.

## Розсилка

Головний адмін → `📢 Розсилка` → ввести текст (HTML-теги підтримуються) → підтвердити. Відправляється з rate-limit 25 повідомлень/сек, лог в БД у таблиці `broadcasts`.

## Локальний запуск

```bash
cd pereviznyk-bot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# заповнити .env
export $(grep -v '^#' .env | xargs)
python bot.py
```

## Bot.log та debugging

- `bot.log` — усі події (INFO+) та помилки.

- Щоб перезапустити бота на Railway — зайти у Deployments → Restart.

## Обмеження / майбутні покращення

Наразі прибране / не зроблено:
- Автоматичне планування на певні години тижня (лише години від створення).
- Веб-панель з аналітикою (тільки Excel-експорт + текстова статистика в боті).
- Автоматичне визначення тонажу/типу авто з існуючих авто-баз.
- Багатомовність (лише українська).

Ці речі можна доточити без переписування — архітектура підтримує.
